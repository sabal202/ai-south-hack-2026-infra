This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Terraform infrastructure for AI Talent Camp hackathon on Cloud.ru Evolution. Provisions an edge/NAT VM (bastion + Traefik + Xray TPROXY) and isolated team VMs in a private subnet. Post-deploy configuration via Ansible. All documentation is in Russian.

## Key Commands

```bash
# All terraform commands run from the environment directory
cd environments/dev

# Format all .tf files (required before commit)
terraform fmt -recursive

# Validate configuration
terraform init
terraform validate

# Plan and apply
terraform plan
terraform apply

# Full format check (CI-style)
terraform fmt -check -recursive

# Ansible (after terraform apply)
cd ansible
ansible-playbook playbooks/edge.yml
ansible-playbook playbooks/team-vms.yml
```

## Architecture

**Two-tier network topology:**
- **Public subnet** (edge VM): single entry point with floating IP, runs Traefik (Docker) for HTTP/HTTPS reverse proxy with TLS passthrough, Xray (systemd) for transparent AI API proxying via TPROXY, and iptables NAT/MASQUERADE + hairpin NAT
- **Private subnet** (team VMs): isolated VMs per team, default route via cloud-init static route to edge VM

**Infrastructure split:**
- **Terraform**: VM provisioning, networking, security groups, SSH key generation
- **Ansible**: all post-deploy configuration (Docker, Traefik, Xray, NAT rules, services)

**Module dependency chain:**
```
data sources (AZ, flavor, disk_type)
  → network → security → edge  → team_vm
                                → team-credentials
                                → ansible inventory
```

Both subnets are created in the network module (no circular dependency without route tables).

## Module Layout

Each module under `modules/` follows: `main.tf`, `variables.tf`, `outputs.tf`, `versions.tf`. The entry point environment is `environments/dev/`.

| Module | Purpose |
|--------|---------|
| `network` | Two subnets (public + private), no VPC resource in evo-terraform |
| `security` | Security groups for edge and team VMs (inline rules, CIDR-based) |
| `edge` | Edge/NAT VM with floating IP, dual network interfaces |
| `team_vm` | Per-team VMs in private subnet with static IPs |
| `team-credentials` | SSH key generation and credential file management |

## Ansible Structure

```
ansible/
├── playbooks/     # site.yml, edge.yml, team-vms.yml, update-services.yml
├── roles/         # common, docker, nat, traefik, xray
├── group_vars/    # all.yml, edge.yml, team_vms.yml
├── inventory/     # hosts.yml (generated by Terraform)
└── templates/     # inventory.yml.tpl (Terraform templatefile)
```

## Important Patterns

- **Provider**: Cloud.ru Evolution (`cloud.ru/cloudru/cloud` v1.6.0), installed via filesystem mirror
- **Auth**: `project_id`, `auth_key_id`, `auth_secret` — passed via `terraform.tfvars` (gitignored)
- **Data sources**: AZ, flavor, disk type looked up dynamically in `environments/dev/main.tf`
- **SSH keys**: 3 key pairs per team (jump/bastion, VM access, GitHub deploy), generated as `tls_private_key` resources
- **Static IPs**: team VMs get deterministic IPs via `cidrhost()` — team-01=10.0.2.11, test=10.0.2.100, orga=10.0.2.101
- **Teams map**: defined in `terraform.tfvars` as a map keyed by team ID (e.g., `"01"`) with `user` and `public_keys` fields
- **Ansible inventory**: auto-generated by Terraform from `ansible/templates/inventory.yml.tpl`

## Cloud.ru Evolution Specifics

- No VPC resource — subnets are top-level resources
- No route table resource — static routes configured via Ansible on team VMs
- Security group rules are inline (`rules {}` blocks), source is CIDR (`remote_ip_prefix`)
- All resources require `availability_zone` block
- Subnet names must be >= 7 characters
- Port ranges format: `"from:to"` or `"any"`
- Compute `image` block sets hostname, username, SSH key (no cloud-init/user_data support)

## Coding Standards

- `terraform fmt` before every commit
- Resources use snake_case: `cloudru_evolution_compute.edge`
- Modules use kebab-case directory names: `team-credentials`
- Comments explain "why", not "what"
- Commit messages follow conventional commits: `feat(edge):`, `fix(xray):`, `docs(quickstart):`
- Branch naming: `feature/`, `fix/`, `docs/`, `refactor/`

## Secrets (gitignored)

The `secrets/` directory contains generated SSH keys and `xray-config.json`. All are gitignored except `.gitkeep`. Never commit these files.
